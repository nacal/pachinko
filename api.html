API Reference

@pachinko/lottery

Pachinko lottery engine — weighted draws, multi-stage pipeline, state management, and simulation.

RNG

createRng(seed: RngSeed): Rng

Create a seeded Xoshiro128** PRNG. Deterministic output for the same seed.

import { createRng } from "@pachinko/lottery";
const rng = createRng({ value: 42 });
rng.next();       // float in [0, 1)
rng.nextInt(100); // integer in [0, 100)

Weighted Selection

weightedSelect(entries, rng): T

Select a random item from weighted entries using linear scan.

createWeightedSelector(entries): WeightedSelector<T>

Pre-compute a selector for repeated weighted draws (binary search).

isHit(probability: ProbabilityInput, rng: Rng): boolean

Test a probability check (e.g., 1/319.68 hit rate).

Machine Definition

defineMachine(config: MachineConfig): MachineSpec

Create a validated machine spec from a declarative config.

import { defineMachine, prob } from "@pachinko/lottery";

const machine = defineMachine({
  id: "my-machine",
  name: "My Machine",
  bonusTypes: {
    kakuhen16R: {
      label: "確変16R",
      rounds: 16,
      nextMode: "kakuhen",
    },
    tsujou: {
      label: "通常10R",
      rounds: 10,
      nextMode: { mode: "jitan", spins: 100 },
    },
  },
  modes: {
    normal: {
      probability: prob(1, 319.68),
      reachRate: 0.03,
      distribution: { kakuhen16R: 60, tsujou: 40 },
    },
    kakuhen: {
      probability: prob(1, 39.96),
      reachRate: 0.3,
      distribution: { kakuhen16R: 70, tsujou: 30 },
    },
    jitan: {
      probability: prob(1, 319.68),
      reachRate: 0.1,
      distribution: { kakuhen16R: 50, tsujou: 50 },
    },
  },
  symbols: ["1", "2", "3", "4", "5", "6", "7"],
  kakuhenSymbols: ["7", "3"],
});

prob(numerator, denominator): ProbabilityInput

Helper to express probability as "1 in X". Returns { totalRange, hits }.

validateConfig(config): ValidationResult

Validate a machine config without throwing.

validateMachineSpec(spec): ValidationResult

Validate a resolved machine spec.

Draw Pipeline

draw(machine, rng, state): DrawResult

Run the full lottery pipeline: outcome → bonus type → reels → next state.

import { draw, createState, createRng, defineMachine, prob } from "@pachinko/lottery";

const rng = createRng({ value: 42 });
const state = createState();
const result = draw(machine, rng, state);

// result.outcome: "oatari" | "koatari" | "hazure"
// result.reels: { left, center, right }
// result.isReach: boolean
// result.nextState: GameState

drawOutcome(machine, rng, state): DrawOutcome

Draw only the outcome (oatari / koatari / hazure).

drawBonusType(machine, rng, mode): BonusType

Select a bonus type from the distribution for the given mode.

State Management

createState(): GameState

Create the initial game state (normal mode, no bonuses).

nextState(state, outcome, bonusType?): GameState

Compute the next state based on draw result. Pure function.

Symbols

drawReels(symbols, outcome, rng, options?): ReelResult

Generate 3-reel results. Jackpots show 3 of a kind; reach shows matching left/right.

standardSymbolSet(labels, kakuhenLabels?): SymbolTable

Create a symbol table from string labels.

Simulation

simulate(config: SimulationConfig): SimulationStats

Run a batch simulation and return aggregate statistics.

simulateStream(config): SimulationRunner

Incremental simulation runner for large trial counts.

@pachinko/rendering

Canvas 2D reel animation renderer with OffscreenCanvas + Worker support.

Public API

createReelRenderer(canvas, config): ReelRenderer

Create a renderer with automatic OffscreenCanvas + Worker detection. Falls back to main-thread rendering if unavailable.

import { createReelRenderer } from "@pachinko/rendering";

const renderer = createReelRenderer(canvas, {
  symbolStrip: [
    { id: "1", label: "1", isKakuhen: false },
    { id: "7", label: "7", isKakuhen: true },
    // ...
  ],
  workerUrl: "/worker-entry.js",
});

renderer.onComplete(() => console.log("Animation complete"));
renderer.spin({ outcome: "oatari", reels, isReach: true });

createInlineReelRenderer(ctx, config): ReelRenderer

Create a main-thread renderer directly from a CanvasRenderingContext2D. No Worker required.

ReelRenderer Interface

MethodDescription
spin(result)Start reel animation for a draw result
onComplete(cb)Register callback for animation completion
onPhaseChange(cb)Register callback for phase transitions
skipToResult()Skip animation and show final result immediately
resize(w, h)Resize the renderer
destroy()Clean up worker and cancel animations

Animation Phases

idle → spinning → stopping-left → stopping-center → stopping-right → result

When isReach === true, the stopping-center phase uses a dramatic slowdown effect.

Configuration

RenderConfig

PropertyTypeDescription
symbolStripSymbolSpec[]Ordered symbol strip for the reels
timing?Partial<TimingConfig>Override animation timing
style?Partial<StyleConfig>Override visual style
workerUrl?stringURL to the worker entry (for OffscreenCanvas mode)

TimingConfig

PropertyDefaultDescription

Constants

Default timing values for all animation phases.

DEFAULT_STYLE: StyleConfig

Default visual style (dark theme with red accent).
