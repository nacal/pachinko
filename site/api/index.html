API Reference

@pachinko/lottery

Pachinko lottery engine — weighted draws, multi-stage pipeline, state management, and simulation.

RNG

createRng(seed: RngSeed): Rng

Create a seeded Xoshiro128** PRNG. Deterministic output for the same seed.

import { createRng } from "@pachinko/lottery";
const rng = createRng({ value: 42 });
rng.next();       // float in [0, 1)
rng.nextInt(100); // integer in [0, 100)

Weighted Selection

weightedSelect(entries, rng): T

Select a random item from weighted entries using linear scan.

createWeightedSelector(entries): WeightedSelector<T>

Pre-compute a selector for repeated weighted draws (binary search).

isHit(probability: ProbabilityInput, rng: Rng): boolean

Test a probability check (e.g., 1/319.68 hit rate).

Machine Definition

defineMachine(config: MachineConfig): MachineSpec

Create a validated machine spec from a declarative config.

import { defineMachine, prob } from "@pachinko/lottery";

const machine = defineMachine({
  id: "my-machine",
  name: "My Machine",
  bonusTypes: {
    kakuhen16R: {
      label: "確変16R",
      rounds: 16,
      nextMode: "kakuhen",
    },
    tsujou: {
      label: "通常10R",
      rounds: 10,
      nextMode: { mode: "jitan", spins: 100 },
    },
  },
  modes: {
    normal: {
      probability: prob(1, 319.68),
      reachRate: 0.03,
      distribution: { kakuhen16R: 60, tsujou: 40 },
    },
    kakuhen: {
      probability: prob(1, 39.96),
      reachRate: 0.3,
      distribution: { kakuhen16R: 70, tsujou: 30 },
    },
    jitan: {
      probability: prob(1, 319.68),
      reachRate: 0.1,
      distribution: { kakuhen16R: 50, tsujou: 50 },
    },
  },
  symbols: ["1", "2", "3", "4", "5", "6", "7"],
  kakuhenSymbols: ["7", "3"],
});

prob(numerator, denominator): ProbabilityInput

Helper to express probability as "1 in X". Returns { totalRange, hits }.

validateConfig(config): ValidationResult

Validate a machine config without throwing.

validateMachineSpec(spec): ValidationResult

Validate a resolved machine spec.

Draw Pipeline

draw(machine, rng, state): DrawResult

Run the full lottery pipeline: outcome → bonus type → reels → next state.

import { draw, createState, createRng, defineMachine, prob } from "@pachinko/lottery";

const rng = createRng({ value: 42 });
const state = createState();
const result = draw(machine, rng, state);

// result.outcome: "oatari" | "koatari" | "hazure"
// result.reels: { left, center, right }
// result.isReach: boolean
// result.nextState: GameState

drawOutcome(machine, rng, state): DrawOutcome

Draw only the outcome (oatari / koatari / hazure).

drawBonusType(machine, rng, mode): BonusType

Select a bonus type from the distribution for the given mode.

State Management

createState(): GameState

Create the initial game state (normal mode, no bonuses).

nextState(state, outcome, bonusType?): GameState

Compute the next state based on draw result. Pure function.

Symbols

drawReels(symbols, outcome, rng, options?): ReelResult

Generate 3-reel results. Jackpots show 3 of a kind; reach shows matching left/right.

standardSymbolSet(labels, kakuhenLabels?): SymbolTable

Create a symbol table from string labels.

Simulation

simulate(config: SimulationConfig): SimulationStats

Run a batch simulation and return aggregate statistics.

simulateStream(config): SimulationRunner

Incremental simulation runner for large trial counts.

Key Types

DrawOutcome"oatari" | "koatari" | "hazure"
DrawResult{ rawValue, outcome, bonusType, reels, isReach, previousState, nextState }

@pachinko/effects

Declarative presentation effects engine — flash, shake, text overlays, and composable effect pipelines.

Engine

createEffectsEngine(canvas, config): EffectsEngine

Create an effects engine that renders on a Canvas overlay. Configure with an array of effect rules.

import { createEffectsEngine, flash, textOverlay, shake, sequence } from "@pachinko/effects";

const engine = createEffectsEngine(overlayCanvas, {
  rules: [
    {
      id: "reach-flash",
      condition: { phase: "reach", isReach: true },
      effects: [flash({ color: "#ff0000", opacity: 0.6, count: 3,
                        timing: { delay: 0, duration: 800 } })],
    },
    {
      id: "oatari",
      condition: { phase: "result", outcome: "oatari" },
      effects: [sequence(
        flash({ color: "#ffd700", opacity: 0.8, count: 5,
                timing: { delay: 0, duration: 1000 } }),
        textOverlay("大当り!", { font: "bold 72px sans-serif", color: "#ffd700",
                                timing: { delay: 0, duration: 2000 } }),
      )],
    },
  ],
});

EffectsEngine Interface

MethodDescription
start(drawResult)Set draw result and reset state
setPhase(phase)Trigger phase-based rule evaluation
setReelStop(position, symbol)Notify individual reel stop for condition matching
tick(now)Render current frame
getShakeOffset()Get current shake offset { x, y } for applying to other canvases
onComplete(cb)Register callback for timeline completion
skipToResult()Skip all effects immediately
resize(w, h)Update canvas dimensions
destroy()Clean up

Primitives

flash(options?): FlashEffect

Screen flash with color, opacity, and repeat count.

textOverlay(text, options?): TextOverlayEffect

Text display with configurable font, position, fade in/out.

backgroundChange(options?): BackgroundChangeEffect

Smooth background color transition.

shake(options?): ShakeEffect

Screen shake with decaying intensity. Offset accessible via getShakeOffset().

fade(options?): FadeEffect

Fade in or fade out with configurable color.

imageOverlay(image, options?): ImageOverlayEffect

Image overlay with position, size, and fade in/out.

custom(renderFn, options?): CustomEffect

Custom render function receiving (ctx, progress, width, height).

Composers

sequence(...effects): SequenceComposite

Run effects one after another.

parallel(...effects): ParallelComposite

Run effects simultaneously.

stagger(delay, ...effects): StaggerComposite

Run effects with staggered start times.

Rule Conditions

PropertyTypeDescription
phaseEffectPhase | EffectPhase[]Match effect phase(s)
outcomeDrawOutcome | DrawOutcome[]Match draw outcome(s)
isReachbooleanMatch reach state
gameModeGameMode | GameMode[]Match game mode(s)
bonusTypeIdstring | string[]Match bonus type ID(s)
reelSymbol{ position, symbolId }Match specific reel stop
consecutiveBonuses{ min?, max? }Match consecutive bonus range
custom(ctx) => booleanCustom predicate function

All fields are AND-combined. Array values are OR-combined. Omitted fields match anything.

Effect Phases

pre-spin → spin-start → pre-reach → reach → post-reach → result

Mapped automatically from @pachinko/rendering ReelPhase via the adapter.

Adapter

connectRenderer(renderer, engine): disconnect

Connect a ReelRenderer to the effects engine. Automatically maps phase changes, reel stops, and manages the animation loop. Returns a disconnect function.

@pachinko/tracker

Session data tracking and Canvas visualization — slump graphs, hit history charts, and data lamp stats panels.

Session Tracker

createSessionTracker(config): SessionTracker

Create a session tracker that records spins, computes statistics, and provides immutable snapshots.

import { createSessionTracker, renderStatsPanel, renderSlumpGraph, renderHitHistory } from "@pachinko/tracker";

const tracker = createSessionTracker({
  ballsPerSpin: 4,
  ballsPerRound: { kakuhen16R: 100, tsujou: 100 },
  specProbability: 319.68,
});

// Record spins
tracker.recordSpin({ outcome: "oatari", bonusType: "kakuhen16R", mode: "normal" });

// Get data
const snap = tracker.snapshot();
const stats = tracker.stats();

SessionTracker Interface

Charts

All chart functions follow the signature (ctx, width, height, data, options?).

renderSlumpGraph(ctx, width, height, data, options?)

Line chart showing net ball count over cumulative spins. Positive area filled green, negative filled red. Zero line highlighted.

renderHitHistory(ctx, width, height, hits, options?)

Bar chart showing rotation count between hits. Bars color-coded by bonus type. Labels show rotation count above each bar.

@pachinko/rendering

Canvas 2D reel animation renderer with OffscreenCanvas + Worker support.

Public API

createReelRenderer(canvas, config): ReelRenderer

Create a renderer with automatic OffscreenCanvas + Worker detection. Falls back to main-thread rendering if unavailable.

import { createReelRenderer } from "@pachinko/rendering";

const renderer = createReelRenderer(canvas, {
  symbolStrip: [
    { id: "1", label: "1", isKakuhen: false },
    { id: "7", label: "7", isKakuhen: true },
    // ...
  ],
  workerUrl: "/worker-entry.js",
});

renderer.onComplete(() => console.log("Animation complete"));
renderer.spin({ outcome: "oatari", reels, isReach: true });

createInlineReelRenderer(ctx, config): ReelRenderer

Create a main-thread renderer directly from a CanvasRenderingContext2D. No Worker required.

ReelRenderer Interface

MethodDescription
spin(result)Start reel animation for a draw result
onComplete(cb)Register callback for animation completion
onPhaseChange(cb)Register callback for phase transitions
onReelStop(cb)Register callback for individual reel stops (left → right → center)
skipToResult()Skip animation and show final result immediately
resize(w, h)Resize the renderer
destroy()Clean up worker and cancel animations

Animation Phases

idle → spinning → stopping-left → stopping-center → stopping-right → result

When isReach === true, the stopping-center phase uses a dramatic slowdown effect.
